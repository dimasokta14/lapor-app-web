# Git Ignore Configuration Guide

## Overview

A comprehensive `.gitignore` file has been created for this project. It prevents unnecessary files from being tracked in the git repository, keeping the repository clean and focused on source code.

---

## What's Ignored

### 1. **Dependencies** (Critical)
```
node_modules/          → All installed packages
package-lock.json      → NPM lock file (generated)
yarn.lock              → Yarn lock file (generated)
pnpm-lock.yaml         → PNPM lock file (generated)
npm-debug.log*         → NPM debug logs
yarn-debug.log*        → Yarn debug logs
yarn-error.log*        → Yarn error logs
```

**Why**: These files are regenerated automatically by running `npm install` or `yarn install`. No need to commit them.

### 2. **Build Outputs** (Critical)
```
dist/                  → Vite production build output
build/                 → Alternative build directory
.next/                 → Next.js build (if used)
out/                   → Build output directory
```

**Why**: These are generated from source code and should never be committed. They're regenerated during deployment.

### 3. **Environment Variables** (Security)
```
.env                   → All environment variables
.env.local             → Local environment overrides
.env.development.local → Development specific
.env.test.local        → Test specific
.env.production.local   → Production specific
```

**Why**: Environment files contain sensitive data (API keys, database passwords, etc.) and should never be committed.

### 4. **IDE & Editor Files** (Cleanliness)
```
.vscode/               → VS Code settings
.idea/                 → IntelliJ IDEA settings
*.swp, *.swo           → Vim swap files
*~                     → Emacs backups
.sublime-project       → Sublime Text project
.sublime-workspace     → Sublime Text workspace
*.iml, *.iws, *.ipr    → IntelliJ files
.project               → Eclipse project
.classpath             → Eclipse classpath
.c9/                   → Cloud9 IDE
.settings/             → Eclipse settings
*.launch               → Eclipse launch configs
```

**Why**: These are user-specific and vary by developer. They should not be shared.

### 5. **Operating System Files** (Cleanliness)
```
.DS_Store              → macOS folder metadata
Thumbs.db              → Windows image cache
.AppleDouble           → macOS file metadata
.LSOverride            → macOS file metadata
._*                    → macOS resource forks
.Spotlight-V100        → macOS search index
.Trashes               → macOS trash folder
```

**Why**: Generated by the OS and vary between developers' machines.

### 6. **Testing & Coverage** (Build Artifacts)
```
.nyc_output/           → NYC coverage output
coverage/              → Test coverage reports
*.lcov                 → LCOV coverage format
.jest_cache/           → Jest cache
.vitest/               → Vitest cache
test-results/          → Playwright test results
cypress/downloads/     → Cypress downloads
cypress/screenshots/   → Cypress screenshots
cypress/videos/        → Cypress videos
blob-report/           → Playwright blob report
playwright-report/     → Playwright report
```

**Why**: These are generated during test runs and don't need version control.

### 7. **Cache & Temporary Files** (Performance)
```
.eslintcache           → ESLint cache
.stylelintcache        → Stylelint cache
.cache/                → Generic cache
.vite/                 → Vite cache
.turbo/                → Turbo cache
.tmp/                  → Temporary files
.temp/                 → Temporary files
*.tmp, *.temp          → Temporary files
*.log                  → Log files
logs/                  → Log directory
pids/                  → Process ID files
*.pid, *.seed          → PID/seed files
```

**Why**: Cache and temporary files are regenerated and shouldn't be committed.

### 8. **Type Definitions** (Generated)
```
*.tsbuildinfo          → TypeScript build info
dist/                  → Compiled output
```

**Why**: Generated during the TypeScript build process.

## How to Use

### Standard Usage
The `.gitignore` file is automatically used by Git. Once created, Git will ignore all files matching the patterns.

```bash
# These commands won't include ignored files
git add .
git status              # Won't show ignored files
git diff                # Won't show ignored files
```

### Verify Ignored Files
To check what files Git is ignoring:

```bash
# List all ignored files
git status --ignored

# Check if a specific file is ignored
git check-ignore -v filename.log

# See why a file is ignored
git check-ignore -v path/to/file
```

### Temporarily Include Ignored Files
To force add an ignored file (not recommended):

```bash
git add -f node_modules/
```

### Remove Already-Committed Files
If you accidentally committed ignored files:

```bash
# Remove from Git (but not from disk)
git rm --cached filename

# Remove directory
git rm -r --cached directory/

# Commit the removal
git commit -m "Stop tracking node_modules"
```

## Environment Variables (.env)

### Creating .env.example
To help other developers, create a `.env.example` file with template variables:

```bash
# .env.example (COMMIT THIS)
VITE_API_URL=https://api.example.com
VITE_API_KEY=your_api_key_here
VITE_APP_NAME=LaporGaruda
VITE_ENABLE_ANALYTICS=true

# Local .env (DO NOT COMMIT)
VITE_API_URL=http://localhost:3000
VITE_API_KEY=dev_key_12345
VITE_APP_NAME=LaporGaruda_Dev
VITE_ENABLE_ANALYTICS=false
```

### Developers Workflow
1. Clone repository
2. Copy `.env.example` to `.env`
3. Update `.env` with local values
4. Run `npm install`
5. Start development with `npm run dev`

## Git Best Practices with .gitignore

### ✅ DO:
- Commit source code
- Commit documentation
- Commit configuration templates (.env.example)
- Commit package.json
- Keep .gitignore in version control

### ❌ DON'T:
- Commit node_modules/
- Commit build outputs (dist/)
- Commit environment files (.env)
- Commit IDE/editor settings (.vscode/.idea)
- Commit sensitive data (API keys, passwords)
- Commit OS-specific files (.DS_Store, Thumbs.db)

## Common Issues & Solutions

### Issue: Node modules taking up space after cloning
**Solution**: Run `npm install` - node_modules will be created locally, not tracked by Git.

### Issue: Accidentally committed .env file
**Solution**:
```bash
# Remove from Git history
git rm --cached .env
git commit -m "Remove .env from tracking"

# Add to .gitignore
echo ".env" >> .gitignore
git add .gitignore
git commit -m "Add .env to gitignore"
```

### Issue: IDE files being tracked
**Solution**: The .gitignore already handles this, but to remove already-tracked files:
```bash
git rm -r --cached .vscode/
git commit -m "Remove .vscode from tracking"
```

### Issue: Build directory being tracked
**Solution**: Remove and re-add .gitignore:
```bash
git rm -r --cached dist/
git commit -m "Remove dist from tracking"
```

## Regular Maintenance

### Monthly Cleanup
```bash
# Remove ignored files from tracking (if accidentally added)
git clean -fd       # Remove untracked files
git clean -fdx      # Also remove ignored files

# Check for large files that shouldn't be committed
git rev-list --all --objects | \
  sed -n $(git rev-list --objects --all | \
  cut -f1 -d' ' | \
  git cat-file --batch-check | \
  grep blob | \
  sort -t' ' -k3 -n | \
  tail -10 | \
  while read hash type size; do \
    echo -n "-e $hash"; \
  done) | \
  cut -d' ' -f2-
```
